# 「我車呢？」行動應用程式 前端開發與設計指引

本手冊為「我車呢？」(wheres_the_car) 專案 Flutter 前端工程的 UI/UX 與商業邏輯開發規範。整體架構以 Material 3 (M3) 作為視覺基準，狀態管理由 `Bloc` 全面接管（禁用 Cubit），路由與頁面狀態保留由 `go_router` 實現。

---

## 1. 核心開發限制與視覺語彙

### 1.1 視覺與介面限制
*   **色調限制**：**全專案 UI 禁用 any 形式的漸層色 (No Gradients)**。所有按鈕、卡片、背景等元件僅可使用 Material 3 配色系統中的單一純色（Solid Colors）或色調變體。
*   **主題配色與深色模式**：**主題色先統一使用 Material 3 的預設顏色 (Baseline Color Scheme)**，並同步配置其對應的 **深色模式 (Dark Mode)** 預設配色。確保系統能依設備環境自動且流暢地切換深淺色主题配色。

### 1.2 狀態與導航架構
*   **狀態管理**：全面採用 `flutter_bloc` 中的 `Bloc` 類別，**嚴禁使用 `Cubit`**。所有資料流與事件（Events）必須在 Bloc 中保持單向流通。
*   **導航切換**：使用 `go_router` 搭配 `StatefulShellRoute`。主要分頁切換時，必須保留其 Scroll 滾動進度與狀態，避免切換分頁時重新讀取。

---

## 2. 磨砂搜尋窗 (Spotlight)

### 2.2 液態玻璃全域搜尋 (Spotlight Search)
*   **視覺風格**：採用 **Liquid Glassmorphism (液態玻璃擬態)**。搜尋框背景為 BackdropFilter 模糊處理的半透明磨砂質感，邊緣搭配高亮細邊線。
*   **互動**：彈出時，搜尋框以外的背景全部做毛玻璃模糊化。點擊搜尋框外部區域，搜尋框會伴隨淡出動畫關閉。
*   **搜尋整合**：整合台鐵、高鐵、捷運、公車。由後端 `/api/embed` 與本地 `sqlite_vector` 進行 1024-dim 向量檢索。

## 3. 主要分頁開發規格

### 3.1 主頁 (Home Grid)
*   **Google Photos 風格隱藏面板**：
    *   主頁面支援**向上拖曳/滑動 (Swipe Up) 手勢**。
    *   往上拉動時，會流暢地從底部拉出伺服器運作狀態及效能數據的即時監控面板。平時狀態下此面板完全隱藏。
*   **首次啟動引導與微動畫 (Onboarding Anim)**：
    *   首次下載安裝並開啟 App 時，主頁會先載入一組預設的 Widget 排版。
    *   **動畫引導機制**：為提示用戶此區域可自由擺放，這組預設 Widget 初始會呈現**灰色 (Greyscale) 且無法點擊**。同時，Widget 卡片會伴隨**微弱的縮放呼吸脈衝動畫 (Pulse Animation)**，並且在網格上方顯示一個**半透明的「手勢長按」動態圖標（模擬手指按壓並產生漣漪波紋）**。
    *   一旦用戶受動畫引導進行「長按」，即會進入編輯模式，並解除灰色鎖定狀態，正式啟用主頁。
*   **隱藏網格編輯模式 (Grid Board)**：
    *   主體為隱藏格線的 4 行網格佈局。長按即可觸發編輯模式。
    *   **邊緣控制槓 (Border Handles)**：進入編輯模式後，**每個 Widget 的四周會亮起一圈細控制槓 (Handles)**，供用戶進行拖拽移動或拉伸調整大小（自由適應 4x4 內的各種格狀網格尺寸）。
*   **主頁 Widget 清單**：
    1.  附近公車站 / 2. 附近台鐵站 (帶台鐵圖標) / 3. 附近高鐵站 (帶高鐵圖標) / 4. 附近捷運站 (帶捷運/線路圖標，桃機捷只顯示系統圖標，高雄輕軌與高捷共用 `AM.svg` 圖標) / 5. 一鍵去某地 / 6. 收藏路線 / 7. 附近自行車。
    8.  **MaaS A/B 導航 Widget**：串接 TDX MaaS API，引導用戶輸入參數（`gc` 時間/價格比偏好、`transit_mode` 運具列表、選填的 `first_mile`/`last_mile` 時間與模式、`arrival`/`depart` 時間以及 `transfer_time` 轉乘區間等），完成後在地圖上呈現導航 Polyline。

## 4. 子頁面與特化地圖模式

### 4.1 公車到站資訊 (Bus ETA Flow)
*   **到站提醒與震動回饋**：
    *   在到站資訊清單中，用戶**長按**（Long Press）任一站點名稱或圈圈，即會在其右側加入一個鬧鐘圖示表示已啟用「到站提醒」。
    *   當公車抵達該站時，裝置必須**持續震動 6.7 秒**以觸發強提醒。
*   **行車進度 Card**：
    *   公車行駛於站間時，以 Card（"台大 -> 台科大"）顯示在該區間上。
    *   Card 下方有一條代表該區間長度的水平線，線上以**黑槓**標示公車即時相對位置，黑槓下方顯示車牌號碼。

### 4.2 公車路線地圖模式
*   **入口**：公車到站資訊頁底部常駐一個**漂浮的長方圓形按鈕 (Floating Stadium Button)**。
*   **內容**：跳轉至地圖，繪製該公車路線的 **Polyline**。所有站點以 `stop.svg` 為 Marker，中間框框顯示即時到站資訊，點擊後跳出對話框顯示站名。
*   **公車位置**：以**方向箭頭**展現：**藍色箭頭**為去程，**綠色箭頭**為返程（箭頭指向對應車輛移動方向）。

## 5. 異常通阻層級 (Alert Hierarchy)

*   **捷運/公車/鐵路**各自在所屬分頁與子頁面中顯示獨立的營運通阻資訊。
*   **主頁彈出規則**：僅在通阻燈號為**紅燈 (嚴重異常)** 時，主頁最上方才會主動彈出 Card 提示。
*   **其他系統頁面彈出規則**：只要達到**黃燈 (一般異常 / 延誤)**，對應的系統分頁與子畫面即必須主動彈出 Card 供用戶警覺。

## 6. 背景即時活動 (Live Activity / Live Update)

當用戶在捷運、公車、台鐵、高鐵中選取特定班次/站點並**退出程式 (進入背景)** 時觸發。

*   **左上方 (主資訊)**：捷運顯示「線路名」與「上一站 -> 下一站」；公車顯示「路線名」；台鐵/高鐵顯示「車次」。
*   **右上方**：預計抵達下一站的時間。
*   **下方 (進度推估)**：使用班表時間推估即時位置，形式比照公車進度線，但**精簡並統一分為三段**（出發、接近、即將到站）。
